Kubectl

kubectl 是用于针对 Kubernetes 集群运行操作管理命令的命令行接口，也就是 kubernetes 的客户端工具。本文对 kubectl 的使用进行总结，深入学习 kubernetes 集群管理操作。

kubectl 语法格式：

$ kubectl [command] [type] [name] [flags]

其中 command,type,name,flags 说明如下：

  • command: 指定要在一个或多个资源进行操作命令，例如 create，get，describe，delete,apply等操作，可以由 $ kubectl --help 获取 command 的所有命令集合。
  • type: 指定资源对象类型，不区分大小写，但可以指定单复数或缩写形式。例如 po Pod Pods pod pods POD PODS 等，但缩写形式不支持大写。
  • name: 指定资源对象的名称，区分大小写。如果省略名称，默认显示所有资源的详细信息。在多个资源对象上执行操作时，可以按资源类型和名称来指定每个资源，或指定一个或多个文件：
    • 指定多个同类型的资源 kubectl get type name01 name02 ...
    • 分别指定多种不同类型的资源 kubectl get type1/name1 type2/name2 type3/name3 ...
    • 使用一个或多个文件指定资源 kubectl get -f file1 -f file2 -f file3 ...
  • flags: 指定可选标志。例如：-o -l -w -s 等来指定一些可选项参数。

常见`flags 如下表：

Output format Description
-o=custom-columns-file=<filename> 自定义输出模板格式打印显示
-o=custom-columns=<spec> 使用逗号分隔打印显示
-o=json 输出json格式的api对象
-o=jsonpath=<template> 打印出自定义的字段
-o=jsonpath-file=<filename> 打印文件定义的字段
-o=name 仅打印资源对象的名称，不输出内容
-o=wide 以纯文档格式输出任何附加的信息
-o=yaml 输出yaml格式的api对象资源信息

一级字段：

  • apiVersion : API 版本
  • kind ：资源类型
  • metadata ：资源元数据
  • spec ：资源期望定义
  • statue ：kubernetes 自动维护的状态

要整理的知识点：

  • apiVersion 都有哪些常见定义，以及每种资源对象的定义
  • kind 都有哪些常见定义，基本作用
  • metadata 都有哪些常见参数定义及作用
  • spec 针对不同资源对象有怎样的定义
