Ansible学习手册

Command Modules

模块(又称任务插件或库插件)是可以从命令行或 playbook 任务中使用的离散代码单元。几乎所有的模块都是采用 key=value 形式的 yaml 文件进行定义参数。

ansible 模块分类：

  • Commands modules
  • Files modules
  • Packaging modules
  • Cloud modules
  • Clustering modules
  • Crypto modules
  • Database modules
  • Inventory modules
  • Messaging modules
  • Monitoring modules
  • Net Tools modules
  • Network modules
  • Notification modules
  • Remote Management modules
  • Source Control modules
  • Storage modules
  • System modules
  • Utilities modules
  • Web Infrastructure modules
  • Windows modules

命令模块

  • command ：远程执行命令
    ansible 默认使用 command 模块，可以在 ansible.cfg 文件中修改默认执行的模块。 具体来看下这个 command 的使用参数:
参数 说明
argv 提供字符串或列表表单，不可同时提供，shell命令行中 -a
chdir 运行命令前先切换到的目录
creates 文件存在则不会执行命令
free_form 自由模式
removes 文件存在则执行命令
stdin 标准输出
warn 警告设置，yes/no

直接命令行举例：

$ ansible nginx -m command -a "chdir=/usr/local pwd"   # 相当于 ansible nginx -a "chdir=/usr/local pwd"
172.16.0.8 | CHANGED | rc=0 >>
/usr/local
$ ansible nginx -m command -a "chdir=/opt/ ls"
$ ansible nginx -m command -a "chdir=/opt/ creates=not-file echo '文件不存在时输出'"   
$ ansible nginx -m command -a "chdir=/opt/ removes=exist-file echo '文件存在时输出'"

yaml 格式举例：

- name: cat command
  command: cat /etc/motd
  register: mymotd

- name: 命令列表
  command: 
  args:
    argv: 
      - echo 11
      - echo 22

- name: 文件不存在则执行，存在则忽略
  command: echo $(date)
  args:
    creates: /opt/test.txt

  • expect ：远程执行命令并返回提示 静默处理，指令不会通过 shell 执行，比如变量，<,>,|,$,&等都不能正确解析。

  • psexec : 远程执行Windows命令

  • raw ：执行低-SSH命令

  • script : 传输脚本到远程并远程执行

  • shell ：远程执行命令

  • telnet ：远程执行Telent命令

$ ansible all -m command -a "ip addr"
$ ansible all -m shell -a "ip addr"
$ ansible all -m raw -a "ip addr"

加密模块：

  • certificate_complete_chain 给定一组不受信任和根证书的完整证书链
  • openssl_certificate 生成或检查openssl证书
  • openssl_csr 生成openssl证书签名请求
  • openssl_dhparam 生成openssl diffie-hellman参数
  • openssl_pkcs12 生成openssl pkcs
  • openssl_privatekey 生成证书私钥
  • openssl_publikey 生成公钥

文件模块：

  • acl - 设置和检索文件ACL信息
  • archive - 创建一个或多个文件或树的压缩存档
  • assemble - 从片段组装配置文件
  • blockinfile - 插入/更新/删除由标记线包围的文本块
  • copy - 将文件复制到远程位置
  • fetch - 从远程节点获取文件
  • file - 设置文件的属性
  • find - 根据特定条件返回文件列表
  • ini_file - 调整INI文件中的设置
  • iso_extract - 从ISO映像中提取文件
  • lineinfile - 管理文本文件中的行
  • patch - 使用GNU补丁工具应用补丁文件
  • replace - 使用反向引用的正则表达式替换文件中特定字符串的所有实例。
  • stat - 检索文件或文件系统状态
  • synchronize - 围绕rsync的包装器，可以快速轻松地在您的剧本中完成常见任务。
  • tempfile - 创建临时文件和目录
  • template - 将文件模板发送到远程服务器
  • unarchive - 在（可选）从本地计算机复制存档后解压缩存档。
  • xattr - 管理用户定义的扩展属性
  • xml - 管理XML文件或字符串的各个部分
