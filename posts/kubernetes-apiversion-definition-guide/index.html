Which Kubernetes apiVersion Should I Use?

Kubernetes Apiversion Definition Guide

Kubernetes 中对资源对象的定义需要一个 apiVersion 字段。当 Kubernetes 有一个更新可供我们使用的版本时 - 在 API 中更新某些内容 - 即 apiVersion 被创建一个新版本。

然而,官方提供 Kubernetes 文档中对 apiVersion 说明却很少。本文将提供了使用哪个版本的备忘单，解释了每个版本，并为您提供了发布的时间表。

最后更新时间为2018年6月4日。请与我联系以报告不准确之处。

apiversion 版本的说明

  • alpha :带有 alpha 字样的 API 版本是 Kubernetes 为开发测试新功能的开发版本。这些版本可能包含错误，并且不保证将来一定可以使用的版本。
  • beta :该 API 版本是测试版本，说明已经通过 alpha 的功能测试，该功能最终将被纳入 Kubernetes。虽然它的工作方式可能会改变，且定义对象的方式可能会完全改变，但该特征本身很可能以某种形式将其变为 Kubernetes。
  • stable:不包含 alpha 或 beta 字样的 API 版本是稳定版本。
  • v1:这是 Kubernetes API 的第一个稳定版本,它包含许多核心对象。
  • apps/v1:apps 是 Kubernetes 中最常见的 API 组，其中包含许多核心对象 v1,它包括与在 Kubernetes 上运行应用程序相关的功能，如 Deployments,RollingUpdates和 ReplicaSets 等。
  • autoscaling/v1:该 API 版本允许根据不同的资源使用指标来自动调整 pod 的期望状态。目前仅支持 CPU 扩展，但未来的 alpha 和 beta 版本将允许根据内存使用情况和其他自定义指标进行扩展.
  • batch/v1:batch API 组包含与批处理和作业类任务（而不是应用类的任务，如无限期运行 Web 服务器）的对象。
  • batch/v1beta1:batch API 组中批处理对象的新功能测试版本，特别是包括允许在特定时间或周期运行作业的 CronJobs。
  • certificates.k8s.io/v1beta1:该 API 版本添加了验证网络证书的功能，以便在群集中进行安全通信.可以在官方文档上阅读更多的内容。
  • extensions/v1beta1:该 API 版本包含许多新的 Kubernetes 常用功能,Deployment，DaemonSet，ReplicaSet 和 Ingresses 都在此版本中进行重大更改。请注意，在 Kubernetes 1.6 版本中，其中一些 extensions 组的资源对象被重新分配到其他具体的 API 组（例如 apps）。当这些对象通过并移出 beta 版本时，期望它们在像 apps/v1 的具体 API 组中。 extensions/v1beta1 已被弃用 - 尝试尽可能使用特定的 API 组，这具体取决于使用的 Kubernetes 集群版本。
  • policy/v1beta1:该 API 版本中增加了设置 pod 中断预算和 pod 安全性新规则的功能。
  • rbac.authorization.k8s.io/v1:该 API 版本包含了 Kubernetes 基于角色访问控制的额外功能。这有助于保护群集,具体查看官方博客文章。
